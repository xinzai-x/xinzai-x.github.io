内存分区

变量在内存分析

普通局部变量

定义形式：在函数里面定义的普通变量

作用范围：离它最近的 {} 之间有效

生命周期：离它最近的 {} 之间有效，离开 {} 的局部变量系统自动回收

存储区域：栈区

普通局部变量不初始化内容不确定

普通局部变量同名 就近原则（同一个{}里的）

普通全局变量

定义形式：定义在函数外边的变量就是普通全局变量

作用范围：

当前源文件都有效，全局变量声明的时候最好也加上extern（用于区别）

其他源文件使用全局变量时，必须加 extern 声明

生命周期：整个 进程 都有效(程序结束的时候全局变量才被释放)

存储区域：全局区

注意事项：

全局变量不初始化内容为 0

如果全局变量要在其他源文件中使用必须在所使用的源文件中加 extern 声明

如果全局变量和局部变量同名 在 {} 中优先使用局部变量

静态局部变量

定义形式：定义局部变量前面必须加 static 修饰

作用范围：离它最近的 {} 之间有效

生命周期：整个进程（程序结束的时候静态局部变量才被释放）

存储区域：全局区

静态局部变量不初始化内容为 0（全局区）

只能被初始化一次（重要）下一次运算依据上一次的结果值

静态全局变量

定义形式：在函数外边定义同时加 static 这样的变量就是静态全局变量

作用范围：当前源文件有效 不能在其他源文件 中使用

生命周期：整个进程(程序结束 静态全局变量才被释放)

存储区域：全局区

静态全局变量不初始化内容为 0

静态全局变量只在当前源文件有效

函数在内存分析

全局函数(普通函数)

其他源文件可以使用全局函数，必须加 extern 声明

静态函数(局部函数)

只能在当前源文件使用，不能在其他源文件使用

注意：如果想在其他源文件调用静态函数，需要将静态函数封装在全局函数中，同时全局函数和静态函数必须是同一个源文件

内存填充

通常用于初始化内存区域，例如将数组的所有元素设置为零或某个特定值

内存比较

逐字节比较从 s1 和 s2 开始的连续 n 个字节，并返回一个整数值来指示比较的结果

内存拷贝

从 src 开始的连续 n 个字节复制到 dest 开始的内存区域，dest 和 src 所指向的内存区域不能重叠。如果需要处理重叠的内存区域，应该使用 memmove 函数
