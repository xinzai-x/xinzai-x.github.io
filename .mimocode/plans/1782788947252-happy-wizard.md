# Plan: Fix External Links in Encrypted Posts

## Goal
Ensure external links in encrypted (password-protected) posts also open in new tabs.

## Problem
The `hexo-blog-encrypt` plugin decrypts content dynamically AFTER `DOMContentLoaded`. The `LinkManager.init()` runs on `DOMContentLoaded`, but at that time encrypted posts only show the password form. After decryption, the plugin replaces innerHTML with decrypted content and fires a `hexo-blog-decrypt` event — but by then `LinkManager` has already run, so the new links don't get `target="_blank"`.

## Solution
Modify `LinkManager` to:
1. Extract link processing into a reusable function
2. Listen for the `hexo-blog-decrypt` event to process links in newly decrypted content

## File to Modify
- `D:\tongbu\Blog\themes\ink\source\js\main.js` (lines 459-469)

## Change
```javascript
const LinkManager = {
  processLinks() {
    document.querySelectorAll('a[href]').forEach(link => {
      if (link.hostname && link.hostname !== window.location.hostname) {
        link.setAttribute('target', '_blank');
        link.setAttribute('rel', 'noopener noreferrer');
      }
    });
  },
  init() {
    this.processLinks();
    // Re-process after encrypted post decryption
    window.addEventListener('hexo-blog-decrypt', () => this.processLinks());
  }
};
```

## Verification
1. Run `hexo generate` and `hexo server`
2. Visit an encrypted post (e.g., `/工具分享/秘密科技/`)
3. Enter password and decrypt
4. Verify external links in decrypted content open in new tabs
