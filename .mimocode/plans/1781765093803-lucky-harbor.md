## 3. Code Block Overflow & Scrollbar Fix

### Root cause analysis:
Two confirmed issues:
1. **Short code blocks are truncated**: The `.highlight` wrapper has `overflow: hidden` (style.css:1148) which clips content. The Hexo-generated code blocks use a `<table>` layout inside `.highlight`. The table's `width: 100%` can constrain content width. The `.highlight .code` has `max-height: 420px` which is correct for long code but the combination with `overflow: hidden` on parent can clip short code.
2. **Scrollbar not at right edge**: `.highlight .code` has `padding: 20px` (style.css:1155, highlight.css:41). The scrollbar renders inside the padding area, leaving a gap between scrollbar and the border.

### Files to modify:
- `source/css/style.css`
- `source/css/highlight.css`

### Changes:

**A. Fix scrollbar positioning (padding issue)**

In `style.css`:
- Line 1129: `.post-content pre` padding → `padding: 20px 0 20px 20px` (remove right padding)
- Line 1155: `.highlight .code` padding → `padding: 20px 0 20px 20px` (remove right padding)

In `highlight.css`:
- Line 41: `.highlight .code` padding → `padding: 20px 0 20px 20px` (remove right padding)

**B. Fix short code truncation**

The `.highlight` wrapper with `overflow: hidden` clips the table content. Change to:
- `style.css:1148`: `.highlight` overflow → `overflow: visible` (or remove `overflow: hidden`)
- The `.highlight .code` already has `overflow-x: auto; overflow-y: auto` which handles scrolling

Actually, the safer fix is: the `.highlight` table can use `min-width: fit-content` so it doesn't constrain, and we remove `overflow: hidden` from `.highlight`:
```css
.highlight {
  overflow: visible;  /* was hidden */
}
.highlight table {
  min-width: fit-content;  /* allow table to be wider than container */
}
```

**C. Add Firefox scrollbar styling**

```css
.highlight .code,
.post-content pre {
  scrollbar-width: thin;
  scrollbar-color: var(--border-secondary) transparent;
}
```

**D. Ensure pre inside highlight doesn't have conflicting styles**

The `.highlight .code pre` already resets padding/border (highlight.css:48-55). The table-cell gutter and code content need to flow naturally.

- The `_config.yml` has `sidebar: right` and `widgets: [recent_posts, categories, tags]`
- This means ALL non-post pages (home, archive, categories, category, tag, page, 404) show the sidebar

### Fix approach:
Modify `layout.ejs` to only show sidebar on specific page types (archive pages could keep it, but user wants it hidden on home and categories pages).

**Option A (Recommended)**: Add a conditional in `layout.ejs` to only show sidebar on post pages and certain specific pages. Since user says "首页、分类页等不需要", we should hide sidebar on: home, categories, category, tag pages. We can keep it on archive and page templates.

**Simplest approach**: Change the condition to only show sidebar on post pages (already handled separately) and archive pages.

### File to modify:
- `layout/layout.ejs` - Line 25: the `else if (theme.sidebar && theme.sidebar !== false)` condition

### Current logic (layout.ejs:10-38):
1. If post page + sidebar + toc → show post with TOC sidebar
2. If post page (no sidebar) → show post without sidebar
3. If NOT post + sidebar enabled → show content + widget sidebar (THIS IS THE PROBLEM)
4. If NOT post + no sidebar → show content only

### New logic:
Only show widget sidebar on post pages (already handled). For ALL other pages, use non-sidebar layout.

Change the condition at line 25 from:
```ejs
<% } else if (theme.sidebar && theme.sidebar !== false) { %>
```
To just:
```ejs
<% } else { %>
```

This removes the widget sidebar from home, archive, categories, category, tag, page, 404, and links pages. The post page TOC sidebar is unaffected (handled separately at line 10).

## 7. Light Mode Accent Color Visibility

### Current state:
- `style.css:28`: `--accent-primary: #b8860b` (dark gold)
- `style.css:29`: `--accent-secondary: #d4a017` (brighter gold)
- `style.css:31`: `--accent-light: rgba(184, 134, 11, 0.1)` (very transparent gold)
- Light background: `--bg-primary: #faf8f5` (warm white)
- `--bg-code-inline: #f0ece5` (slightly darker warm white)

### Problem:
On warm white (#faf8f5) background, the gold accent (#b8860b) is readable but:
- `--accent-light: rgba(184, 134, 11, 0.1)` is too transparent, making border highlights and inline code backgrounds barely visible
- Inline code has `background-color: var(--bg-code-inline)` (#f0ece5) with `color: var(--accent-primary)` (#b8860b) - contrast ratio ~3.5:1, borderline

### Fix:
- `style.css:31`: Increase opacity of `--accent-light` from 0.1 to 0.15 or 0.18
- Consider making inline code more visible: either darker background or stronger text color
- Optionally: darken `--accent-primary` slightly for better contrast

### Files to modify:
- `source/css/style.css`

### CSS changes:
```css
/* Light theme variables - increase accent visibility */
--accent-primary: #9a7209;  /* was #b8860b, slightly darker for better contrast */
--accent-light: rgba(184, 134, 11, 0.15);  /* was 0.1, more visible */
--bg-code-inline: #ebe5da;  /* was #f0ece5, slightly darker for contrast */
```
